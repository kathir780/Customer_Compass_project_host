About the Project

Customer Compass is a comprehensive data analytics project built to analyze customer segmentation and behavior across time. The goal was to replicate real-world business challenges and solve them using industry-level tools and techniques.

Business Objective: The core objective is to understand how customer segments evolve and what triggers those changes—helping businesses increase retention, reduce churn, and boost sales by targeting the right segments at the right time.

Unlike basic case studies, this project was built entirely from scratch. I designed the ER model, created a 3NF-compliant database schema, generated mock data using Python's Faker library, and visualized the results. Every component—from schema design to insight generation—was engineered with practical use cases in mind.

Database Design

The foundation of the project is a well-normalized relational database. The schema is designed in 3rd Normal Form (3NF) to reduce redundancy and ensure data integrity. Here's a breakdown of key entities:

  • Customers: Contains personal and demographic details of users.
  • Segments: Defines different customer segments based on behavior and preferences (e.g., Loyal, At Risk, High Value).
  • Customer_Segments_History: Tracks changes in customer segments over time to understand behavioral transitions.
  • Products: Includes product metadata like category, price, and popularity.
  • Orders & Order_Items: Captures purchase transactions and itemized order data.
  • Order_Address: Contains shipping location details for geospatial analysis.
  • Payments: Stores payment mode, success status, and related info.

Each relationship was carefully planned and normalized to handle large datasets efficiently. Temporal logic was applied to monitor behavior trends over time.
